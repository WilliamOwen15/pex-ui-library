# Container Component

A foundational layout component that provides responsive width constraints, vertical padding, and background styling. Inspired by Emre Turan's "Shadcn Container" pattern and adapted for the Glide design system.

## Standard Container

The most common usage with default max-width constraints (max-w-7xl). Perfect for dashboard content, forms, and general page layouts.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Content to render inside container (used for single-column layouts) |
| `variant` | `"default" \| "narrow" \| "fullMobile" \| "full"` | `"default"` | Width constraint style |
| `padding` | `"none" \| "sm" \| "md" \| "lg" \| "xl"` | `"md"` | Vertical padding only (py-*) - horizontal padding is handled by variant |
| `background` | `"none" \| "card" \| "accent" \| "dark" \| "image"` | `"none"` | Background style |
| `backgroundImageUrl` | `string` | - | Background image URL (only used when background="image") |
| `title` | `string` | - | Optional section title rendered with Title component |
| `description` | `string` | - | Optional section description (rendered as subtitle in Title) |
| `onClick` | `() => void` | - | Click handler - makes container a button element with keyboard support |
| `as` | `"div" \| "section" \| "article" \| "main"` | `"div"` | HTML element to render (overridden to "button" when onClick is provided) |
| `columns` | `"1:1" \| "1:2" \| "2:1"` | - | Two-column layout ratio (requires leftColumn and rightColumn props) |
| `leftColumn` | `ReactNode` | - | Content for left column (only used when columns is set) |
| `rightColumn` | `ReactNode` | - | Content for right column (only used when columns is set) |
| `className` | `string` | - | Additional CSS classes merged with variant styles |

## Variant Details

### Width Constraints

- **default**: `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8` - Standard dashboard/app layout (1280px max)
- **narrow**: `max-w-3xl mx-auto px-4 sm:px-6` - Reading-optimized width (768px max)
- **fullMobile**: `w-full md:max-w-7xl md:mx-auto md:px-8` - Full-width on mobile, constrained on desktop
- **full**: `w-full` - Always edge-to-edge

### Padding Scale

- **none**: `py-0` - No vertical padding
- **sm**: `py-4` - 16px vertical padding
- **md**: `py-8` - 32px vertical padding (default)
- **lg**: `py-12` - 48px vertical padding
- **xl**: `py-16` - 64px vertical padding

## Usage Guidelines

### When to Use Each Variant

**default**: Most common use case for app layouts, dashboards, forms, and general content. Provides comfortable reading width while utilizing screen space efficiently.

**narrow**: Blog posts, articles, documentation, and any text-heavy content where readability is the primary concern. Optimal line length improves comprehension.

**fullMobile**: Hero sections, promotional banners, and content that benefits from maximum impact on mobile while remaining manageable on desktop.

**full**: Navigation bars, footers, dividers, and backgrounds that should span the entire viewport. Often combined with nested max-width containers for content.

### When to Use Column Layouts

**1:1 (Equal Columns)**: Feature comparisons, before/after sections, balanced content areas, dual CTAs, or any side-by-side content with equal importance.

**1:2 (Narrow Left, Wide Right)**: Sidebar navigation with main content, filter panel with results, table of contents with article content, or any layout where supplementary information accompanies primary content.

**2:1 (Wide Left, Narrow Right)**: Article with related links, main content with advertisements, featured content with supporting info, or layouts where primary content needs emphasis with secondary context nearby.

All column layouts automatically stack vertically on mobile devices for optimal mobile experience.

### Accessibility

- Uses semantic HTML via the `as` prop for proper document structure
- When `onClick` is provided, automatically becomes a `<button>` element
- Includes keyboard support (Enter and Space keys) for clickable containers
- Supports all standard HTML attributes via props spreading
- Color contrast is maintained across all background variants

### Best Practices

1. **Nesting**: Avoid nesting containers with the same variant. Outer containers should provide the constraint.
2. **Background Images**: Use `background="image"` with `backgroundImageUrl` for decorative backgrounds only. For content images, use Next.js Image component.
3. **Clickable Containers**: Only use `onClick` for entire-section interactions. For specific buttons, use Button components inside the container.
4. **Padding**: Use the padding prop for vertical spacing. Horizontal padding is automatically handled by variants.
5. **Title Integration**: Let the Container handle section headers via `title` and `description` props rather than manually adding Title components.

## Performance Notes

**Background Images**: The `background="image"` variant uses CSS `background-image` for simplicity and flexibility. For optimized images with Next.js automatic optimization, use a transparent background container with Next.js `<Image>` components as children instead.
