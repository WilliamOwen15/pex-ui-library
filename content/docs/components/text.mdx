# Text Component

A foundational typography component that provides comprehensive text styling control. Supports semantic HTML rendering, preset variants based on shadcn/ui patterns, and extensive customization options for size, weight, alignment, color, and text transformations.

## Basic Usage

The simplest usage with default paragraph styling. The component renders a `<p>` element by default.

<ComponentPreview
  name="Basic Paragraph"
  code={`<Text variant="p">
  This is a standard paragraph with proper line height and spacing. The Text
  component handles all typography needs with a clean, consistent API.
</Text>`}
>
  <Text variant="p">
    This is a standard paragraph with proper line height and spacing. The Text
    component handles all typography needs with a clean, consistent API.
  </Text>
</ComponentPreview>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | Text content to render |
| `as` | `"h1" \| "h2" \| "h3" \| "h4" \| "h5" \| "h6" \| "p" \| "span" \| "div" \| "label" \| "code"` | Auto | HTML element to render. Defaults based on variant (h1→h1, h2→h2, small→span, etc.) |
| `variant` | `"h1" \| "h2" \| "h3" \| "h4" \| "p" \| "lead" \| "large" \| "small" \| "muted" \| "code"` | `"p"` | Preset typography style from shadcn/ui patterns |
| `size` | `"xs" \| "sm" \| "base" \| "lg" \| "xl" \| "2xl" \| "3xl" \| "4xl"` | - | Override text size (12px to 36px) |
| `weight` | `"light" \| "normal" \| "medium" \| "semibold" \| "bold" \| "extrabold"` | - | Font weight (300 to 800) |
| `align` | `"left" \| "center" \| "right" \| "justify"` | `"left"` | Text alignment |
| `color` | `"default" \| "muted" \| "primary" \| "destructive" \| "success" \| "warning"` | `"default"` | Semantic color variant |
| `transform` | `"none" \| "uppercase" \| "lowercase" \| "capitalize"` | `"none"` | Text capitalization |
| `truncate` | `boolean` | `false` | Single-line truncation with ellipsis |
| `clamp` | `1 \| 2 \| 3 \| 4 \| 5 \| 6` | - | Multi-line truncation (takes precedence over truncate) |
| `className` | `string` | - | Additional CSS classes merged with variant styles |

## Variant Details

### Preset Variants

- **h1**: `scroll-m-20 text-4xl font-extrabold tracking-tight` - Largest heading
- **h2**: `scroll-m-20 border-b pb-2 text-3xl font-semibold tracking-tight` - Secondary heading with border
- **h3**: `scroll-m-20 text-2xl font-semibold tracking-tight` - Tertiary heading
- **h4**: `scroll-m-20 text-xl font-semibold tracking-tight` - Smallest heading
- **p**: `leading-7` - Body paragraph with optimal line height
- **lead**: `text-xl text-muted-foreground` - Introductory text
- **large**: `text-lg font-semibold` - Emphasized text
- **small**: `text-sm font-medium leading-none` - Compact text
- **muted**: `text-sm text-muted-foreground` - Deemphasized text
- **code**: `relative rounded bg-muted px-[0.3rem] py-[0.2rem] font-mono text-sm font-semibold` - Inline code

## Usage Guidelines

### When to Use Each Variant

**h1-h4**: Page titles, section headings, and content hierarchy. Always maintain proper heading order (h1 → h2 → h3 → h4) for accessibility. Use the `as` prop to override rendering when you need h1 styling but it's not semantically an h1.

**p**: Standard body text, descriptions, and content paragraphs. The default variant for most text content.

**lead**: Opening paragraphs, introductions, and callouts that need more emphasis than regular body text.

**large**: Emphasized statements, important information, or when you need slightly larger text without full heading weight.

**small**: Labels, captions, metadata, helper text, and any secondary information that should be compact.

**muted**: Timestamps, supporting details, placeholder text, and any content that should be visually deemphasized.

**code**: Inline code snippets, commands, file names, and technical terms that need monospace styling.

### Combining Props

Props can be combined for precise control. When props conflict with variant styles, the explicit prop takes precedence:

```tsx
{/* Heading style but smaller size */}
<Text variant="h1" size="lg">Custom Sized Heading</Text>

{/* Small text but bold weight */}
<Text variant="small" weight="bold">Bold Label</Text>

{/* Muted color with different size */}
<Text variant="muted" size="base">Larger Muted Text</Text>
```

### Accessibility

**Heading Hierarchy**: Always use headings in order (h1 → h2 → h3 → h4). Never skip levels. Use the `as` prop when you need h1 styling but it's not semantically correct:

```tsx
<Text variant="h1" as="h1">Page Title</Text>
<Text variant="h2" as="h2">Section</Text>
<Text variant="h1" as="p">Styled like h1 but not a heading</Text>
```

**Color Contrast**: All color variants maintain WCAG AA contrast ratios (4.5:1 for normal text, 3:1 for large text). The `muted` color variant may not meet contrast requirements for essential content.

**Form Labels**: Always use the `<label>` element for form inputs with proper `htmlFor` attributes:

```tsx
<Text as="label" variant="small" htmlFor="email">
  Email Address
</Text>
<input id="email" type="email" />
```

**Reading Length**: For optimal readability, keep line length between 50-75 characters. Use the Container component's `narrow` variant for text-heavy content.

### Best Practices

1. **Semantic HTML First**: Choose the `as` prop based on document structure, not visual appearance. Let the `variant` prop handle styling.

2. **Consistent Hierarchy**: Maintain consistent visual hierarchy throughout your application. Don't mix h1 styling for multiple different purposes.

3. **Avoid Over-Customization**: Use preset variants when possible. They're designed to work together as a system.

4. **Truncation**: Test truncated text at various viewport sizes. Ensure critical information isn't hidden by truncation.

5. **Responsive Typography**: For responsive text sizes, use Tailwind responsive prefixes in className rather than JavaScript:
   ```tsx
   <Text className="text-base md:text-lg lg:text-xl">
     Responsive Text
   </Text>
   ```

6. **Performance**: The Text component is pure CSS with no runtime overhead. All styling is compiled at build time.

## Performance Notes

**Zero Runtime Overhead**: All styling is compiled to CSS classes at build time. No JavaScript runs when the component renders.

**CSS-Only Truncation**: Truncation uses pure CSS (`text-overflow` and `line-clamp`), not JavaScript measurement or manipulation.

**Tree-Shaking**: Unused variants are removed during build optimization. Only the classes you use end up in your final bundle.

**Polymorphic Typing**: TypeScript fully types the component based on the `as` prop, providing autocomplete for element-specific attributes with no runtime cost.
