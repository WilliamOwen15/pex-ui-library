# Tabs Container Component

A powerful tabs component that extends the standard Radix UI Tabs with drag-and-drop reordering, conditional visibility, icon and badge support, and visual separators between tab triggers. Built for the Glide design system with full accessibility support.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tabs` | `TabItem[]` | **Required** | Array of tab items with id, label, content, and optional metadata |
| `defaultValue` | `string \| number` | First tab ID | Initially active tab (uncontrolled mode) |
| `value` | `string \| number` | - | Active tab (controlled mode) |
| `onValueChange` | `(value: string \| number) => void` | - | Callback when active tab changes |
| `onTabsReorder` | `(tabs: TabItem[]) => void` | - | Callback when tabs are reordered via drag-and-drop |
| `variant` | `"default" \| "line"` | `"default"` | Tab trigger style variant |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Tab list orientation |
| `draggable` | `boolean` | `false` | Enable drag-and-drop reordering |
| `showSeparators` | `boolean` | `false` | Show visual separators between tab triggers |
| `separatorClassName` | `string` | - | Custom className for separators |
| `containerVariant` | `"default" \| "narrow" \| "fullMobile" \| "full"` | `"default"` | Container width constraint |
| `padding` | `"none" \| "sm" \| "md" \| "lg" \| "xl"` | `"md"` | Vertical padding |
| `background` | `"none" \| "card" \| "accent" \| "dark"` | `"none"` | Background style |
| `as` | `"div" \| "section" \| "article"` | `"div"` | HTML element to render |
| `className` | `string` | - | Additional CSS classes |

### TabItem Interface

| Property | Type | Description |
|----------|------|-------------|
| `id` | `string \| number` | Unique identifier for the tab |
| `label` | `string` | Display text for tab trigger |
| `content` | `React.ReactNode` | Tab content to display |
| `icon` | `React.ReactNode` | Optional icon (displayed before label) |
| `badge` | `string \| number` | Optional badge count or text |
| `disabled` | `boolean` | Disable tab interaction and drag |
| `visible` | `boolean \| ((tab: TabItem, allTabs: TabItem[]) => boolean)` | Visibility condition (boolean or callback) |
| `ariaLabel` | `string` | Custom ARIA label for accessibility |

## Variant Details

### Tab Variants

- **default**: Tabs with subtle background (muted style)
- **line**: Minimal tabs with underline indicator

### Container Variants

- **default**: Standard max-width constraint (1280px)
- **narrow**: Reading-optimized width (768px)
- **fullMobile**: Edge-to-edge on mobile, constrained on desktop
- **full**: Always full width

### Padding Scale

- **none**: No vertical padding
- **sm**: 16px vertical padding
- **md**: 32px vertical padding (default)
- **lg**: 48px vertical padding
- **xl**: 64px vertical padding

## Usage Guidelines

### When to Use TabsContainer vs Tabs

**Use TabsContainer when:**
- You need drag-and-drop reordering
- Tabs should be conditionally visible
- You want icons and badges on tabs
- You need visual separators between tabs
- You want container-level styling control

**Use standard Tabs when:**
- You have a simple, static tab layout
- You need maximum flexibility with custom markup
- You don't need the enhanced features

### Visibility Logic Best Practices

1. **Boolean Visibility**: Use for simple show/hide based on flags
   ```tsx
   visible: isAdmin
   ```

2. **Callback Visibility**: Use for complex logic
   ```tsx
   visible: (tab, allTabs) => userHasPermission(tab.id)
   ```

3. **Always Provide Fallback**: If all tabs might be hidden, show an empty state

### Drag-and-Drop UX Patterns

- Enable `draggable` only when users benefit from custom ordering
- Combine with `onTabsReorder` to persist tab order
- Use visual separators with draggable tabs for clearer boundaries
- Disabled tabs cannot be dragged

### Performance Considerations

- Visibility callbacks are memoized and only re-evaluated when `tabs` changes
- Tab content is unmounted when tab is hidden (not just hidden with CSS)
- Drag-and-drop uses requestAnimationFrame for smooth animations

## Accessibility

### Keyboard Navigation

- **Tab**: Focus into tabs, move focus within tab content
- **Arrow Keys**: Navigate between tab triggers
- **Enter/Space**: Activate focused tab (or initiate drag when draggable)
- **Escape**: Cancel drag operation

### Screen Reader Support

- All tabs have proper ARIA labels and roles
- Tab state changes are announced
- Drag-and-drop operations are announced
- Custom `ariaLabel` prop for descriptive labels

### Focus Management

- Focus is maintained during drag operations
- Focus moves to first visible tab if active tab becomes hidden
- Focus indicators are clearly visible

## Best Practices

1. **Tab Count**: Keep to 3-7 visible tabs for optimal usability. Use visibility conditions to hide less-used tabs.

2. **Content Length**: Each tab's content should be self-contained. Avoid excessive scrolling within tabs.

3. **Drag-and-Drop**: Only enable when users need custom ordering. Most interfaces work fine with static tab order.

4. **Icons**: Use icons to reinforce tab meaning, not as the sole identifier. Always include text labels.

5. **Badges**: Use for counts (notifications, items) or status indicators. Keep badge text short (1-3 characters).

6. **Visibility**: Hidden tabs are removed from the DOM. Don't rely on hidden tab content being rendered.

7. **Separators**: Use sparingly. They work best with the default variant and when you have 4+ tabs.

8. **Controlled vs Uncontrolled**: Use controlled mode (`value` + `onValueChange`) when tab state is managed externally. Use uncontrolled mode (`defaultValue`) for simpler cases.

## Migration from Standard Tabs

```tsx
// Before (standard Tabs)
<Tabs defaultValue="tab1">
  <TabsList>
    <TabsTrigger value="tab1">Tab 1</TabsTrigger>
    <TabsTrigger value="tab2">Tab 2</TabsTrigger>
  </TabsList>
  <TabsContent value="tab1">Content 1</TabsContent>
  <TabsContent value="tab2">Content 2</TabsContent>
</Tabs>

// After (TabsContainer)
<TabsContainer
  tabs={[
    { id: 'tab1', label: 'Tab 1', content: <div>Content 1</div> },
    { id: 'tab2', label: 'Tab 2', content: <div>Content 2</div> }
  ]}
/>
```

The TabsContainer simplifies the API while adding powerful features. The standard Tabs component remains available for maximum flexibility.
