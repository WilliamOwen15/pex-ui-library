import { ComponentPreview } from "@/components/docs";

A component to highlight important text on the screen. It supports multiple mood variants and can include an optional icon to enhance the visual hierarchy.

## Basic Usage

A simple hint without an icon:

<ComponentPreview
  name="Basic Hint"
  code={`<Hint>
  This is a default hint message that provides helpful information to users.
</Hint>`}
>
  <Hint>
    This is a default hint message that provides helpful information to users.
  </Hint>
</ComponentPreview>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "neutral" \| "success" \| "warning" \| "danger"` | `"default"` | The mood variant of the hint |
| `icon` | `React.ReactNode` | `undefined` | Optional icon to display at the start of the hint |
| `className` | `string` | `undefined` | Additional CSS classes to apply |
| `children` | `React.ReactNode` | - | The content to display in the hint |

The component also accepts all standard HTML `div` attributes.

## Accessibility

- The component uses `role="note"` for proper semantic meaning
- Icons are marked with `aria-hidden="true"` to avoid redundant screen reader announcements
- Color is not the only indicator of meaning - icons and text provide context

## Best Practices

1. **Choose the right variant**: Match the variant to the message type for consistency
2. **Use icons consistently**: If you use icons, use them across all hints of the same type
3. **Keep messages concise**: Hints should be brief and to the point
4. **Don't overuse**: Too many hints can overwhelm users - use sparingly
5. **Combine with actions**: For warnings and errors, consider adding actionable next steps
