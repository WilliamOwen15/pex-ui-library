## Introduction

The Big Number component displays numerical values in a large, prominent format, ideal for showing top-level reporting numbers, KPIs, and statistics. Use it to create impactful dashboards and data visualizations.

**Key Features:**
- Display large numbers with labels and descriptions
- Support for icons (Lucide React)
- Multiple variants: card, outline, ghost, minimal
- Responsive grid layouts
- Interactive with href and onClick support
- Prefix and suffix formatting (e.g., "$", "%", "K")
- Custom render functions for advanced formatting
- Fully accessible with ARIA labels

## Custom Rendering

Use render functions to format values dynamically.

<ComponentPreview>
  <CustomRenderExample />
</ComponentPreview>

```tsx
<BigNumberCollection
  columns={3}
  items={[
    { value: 847000, label: "Revenue", icon: <Users className="h-5 w-5" /> },
    { value: 23, label: "Active Groups", icon: <Calendar className="h-5 w-5" /> },
    { value: 92, label: "Satisfaction Score", icon: <ThumbsUp className="h-5 w-5" /> },
  ]}
  renderValue={(item) => {
    if (typeof item.value === "number" && item.value > 1000) {
      return (
        <div className="text-3xl sm:text-4xl font-bold tabular-nums tracking-tight">
          ${(item.value / 1000).toFixed(0)}K
        </div>
      );
    }
    if (item.label === "Satisfaction Score") {
      return (
        <div className="text-3xl sm:text-4xl font-bold tabular-nums tracking-tight">
          {item.value}/100
        </div>
      );
    }
    return (
      <div className="text-3xl sm:text-4xl font-bold tabular-nums tracking-tight">
        {item.value}
      </div>
    );
  }}
/>
```

## Layout Options

Control the grid layout with the `columns` prop:

- `1` - Single column
- `2` - Two columns (responsive)
- `3` - Three columns (responsive)
- `4` - Four columns (responsive)
- `auto` - Automatic responsive grid (1 → 2 → 3 → 4 based on screen size)

```tsx
<BigNumberCollection columns={4} items={[...]} />
```

## Alignment

Align content to the left (default), center, or right.

## Accessibility

The Big Number component follows accessibility best practices:

- **Semantic HTML**: Uses `<a>` for links, `<button>` for interactions, and `<article>` or `<div>` for static content
- **ARIA Labels**: Automatically generates descriptive labels (e.g., "Total Attendance: 847")
- **Keyboard Navigation**: Full keyboard support with Tab, Enter, and Space keys
- **Screen Readers**: Proper announcement of values, labels, and descriptions
- **Color Contrast**: All text meets WCAG AA standards
- **Focus Indicators**: Visible focus states for interactive elements

### Custom ARIA Labels

```tsx
<BigNumberCollection
  items={[
    {
      value: 847,
      label: "Total Attendance",
      ariaLabel: "Total group attendance this month is 847 people",
    },
  ]}
/>
```

## Best Practices

1. **Keep labels concise**: Use short, descriptive labels (1-3 words)
2. **Use appropriate sizes**: Match size to importance (xl for hero metrics, sm for supporting data)
3. **Group related metrics**: Use collections to show related KPIs together
4. **Provide context**: Add descriptions to explain what the numbers mean
5. **Use icons wisely**: Icons should add clarity, not clutter
6. **Format large numbers**: Use K, M, B suffixes or custom formatting for readability
7. **Make it interactive**: Add href or onClick for drill-down functionality
8. **Consider color meaning**: Use gradient or primary variants for positive metrics
9. **Test responsive behavior**: Verify layout works at all screen sizes
10. **Ensure accessibility**: Always provide meaningful aria-labels
