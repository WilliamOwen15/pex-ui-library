# Card Collection Component

A highly versatile Card Collection component that replicates Glide Apps' no-code card functionality. Perfect for displaying grids or lists of items with images, titles, descriptions, and metadata.

## Spacing & Visual Design

The component uses a carefully designed spacing system for optimal visual hierarchy and readability:

### Spacing Scale

- **Card sections**: 32px gap between major sections (image, header, content, footer)
- **Internal content**: 12px gap between title, description, and meta elements
- **Grid gaps**:
  - Card variant: 24px between cards for generous breathing room
  - Minimal variant: 16px for a more compact layout
  - Horizontal orientation: Automatically adjusts to 12px (card) or 8px (minimal)

### Typography Improvements

- Titles use `tracking-tight` and `leading-snug` for better optical balance
- Increased spacing between text elements improves scanability
- Consistent line heights maintain visual rhythm across all card sizes

### Variant Differences

The spacing system differentiates between variants to provide appropriate visual weight:

- **Card variant** (bordered with shadows): More generous spacing matches the visual prominence
- **Minimal variant** (no borders): Tighter spacing suitable for list views and compact layouts

These spacing improvements ensure cards look professional and polished while maintaining excellent readability across all screen sizes.

## Props

### CardCollection

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `CardCollectionItem[]` | - | Array of items to display (required) |
| `size` | `"xs" \| "s" \| "m" \| "l" \| "xl"` | `"m"` | Grid column density |
| `variant` | `"card" \| "minimal"` | `"card"` | Card style with or without borders/shadows |
| `orientation` | `"vertical" \| "horizontal"` | `"vertical"` | Grid layout or list view |
| `aspectRatio` | `"square" \| "video" \| "portrait"` | `"square"` | Image aspect ratio |
| `imageShape` | `"square" \| "circle"` | `"square"` | Image shape (uses Avatar for circles) |
| `titleStyle` | `"simple" \| "bold"` | `"simple"` | Title font weight |
| `limit` | `number` | - | Limit number of displayed items |
| `className` | `string` | - | Additional CSS classes |
| `onItemClick` | `(item, index) => void` | - | Click handler for items |
| `metaPosition` | `"header" \| "footer" \| "above-title" \| "below-title"` | `"above-title"` | Where to display meta text |
| `renderMeta` | `(meta, item) => ReactNode` | - | Custom meta renderer (e.g., for Badges) |
| `renderAction` | `(item, index) => ReactNode` | - | Render action content per item |
| `renderFooter` | `(item, index) => ReactNode` | - | Render footer content per item |
| `showSeparators` | `boolean` | `false` | Show visual separators between sections |
| `itemAs` | `"article" \| "div"` | `"article"` | Semantic HTML element for cards |
| `renderItem` | `(item, index) => ReactNode` | - | Custom render function for full control |
| `clampTitle` | `boolean \| number` | - | Clamp title to lines (true = 2 lines, number = custom) |
| `clampDescription` | `boolean \| number` | - | Clamp description to lines (true = 3 lines, number = custom) |

### CardCollectionItem

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `title` | `string` | Yes | Item title |
| `description` | `string` | No | Item description |
| `image` | `string` | No | Image URL |
| `meta` | `string` | No | Metadata text |
| `id` | `string \| number` | No | Unique identifier (improves key generation) |
| `href` | `string` | No | Makes card a navigational link |
| `action` | `ReactNode` | No | Per-item action content (overrides renderAction) |
| `footer` | `ReactNode` | No | Per-item footer content (overrides renderFooter) |
| `metaPosition` | `"header" \| "footer" \| "above-title" \| "below-title"` | No | Per-item meta position override |
| `ariaLabel` | `string` | No | Custom accessibility label |

## Accessibility

The Card Collection component follows accessibility best practices:

- Uses semantic HTML (`<article>` by default for cards)
- Provides proper ARIA labels for screen readers
- Supports keyboard navigation (Tab, Enter, Space)
- Uses shadcn Avatar component for circular images with fallback support
- Maintains focus indicators for interactive elements
- Converts to semantic `<a>` tags when `href` is provided

## Migration Note

All existing code continues to work without any changes. The new features are purely additive and opt-in. Your current implementations will maintain exact visual parity with previous versions.

## Size Breakdown

- `xs`: Up to 6 columns on extra-large screens (densest)
- `s`: Up to 5 columns on large screens
- `m`: Up to 4 columns on large screens (default)
- `l`: Up to 3 columns on medium screens
- `xl`: Maximum 2 columns (sparsest)

All sizes are responsive and adapt to smaller screens automatically.
