# Separator Component

A simple, accessible separator component for visually dividing content sections. Built on Radix UI primitives with customizable orientation and styling.

The most common usage is a horizontal separator to divide content vertically.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | The orientation of the separator |
| `decorative` | `boolean` | `true` | When true, the separator is purely visual and ignored by screen readers |
| `className` | `string` | - | Additional CSS classes for custom styling |

## Accessibility

- By default, the separator has `decorative={true}`, making it purely visual and ignored by assistive technologies
- Set `decorative={false}` if the separator has semantic meaning and should be announced to screen readers
- The component uses Radix UI's Separator primitive for proper ARIA attributes
- Horizontal separators have `role="separator"` with `aria-orientation="horizontal"`
- Vertical separators have `role="separator"` with `aria-orientation="vertical"`

## Usage Guidelines

### When to Use

- Dividing sections of content within a page or component
- Separating list items or menu options
- Creating visual hierarchy in cards or panels
- Organizing form sections
- Separating header/footer from main content

### Best Practices

1. **Spacing**: Always provide adequate spacing around separators using margin classes (e.g., `my-4`, `mx-2`)
2. **Vertical Height**: When using vertical separators, ensure the parent container has a defined height
3. **Semantic Meaning**: Only set `decorative={false}` if the separator conveys important structural information
4. **Color Contrast**: The default separator uses the `border` color token which adjusts with theme
5. **Thickness**: Keep separators subtle (1-2px) unless emphasis is needed
