## Props

The Textarea component accepts all standard HTML textarea attributes, plus:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes merged with default styles |
| `disabled` | `boolean` | `false` | Disables the textarea |
| `readOnly` | `boolean` | `false` | Makes the textarea read-only |
| `placeholder` | `string` | - | Placeholder text when empty |
| `aria-invalid` | `"true" \| "false"` | - | Indicates validation state for accessibility |
| `rows` | `number` | - | Initial number of visible text lines |
| `maxLength` | `number` | - | Maximum number of characters allowed |
| `value` | `string` | - | Controlled value (requires onChange) |
| `defaultValue` | `string` | - | Initial value for uncontrolled usage |

All standard textarea props are supported including `onChange`, `onBlur`, `onFocus`, `name`, `id`, etc.

## Features

### Auto-Resizing

The textarea automatically adjusts its height based on content using the `field-sizing-content` CSS property. This provides a better user experience without requiring JavaScript.

### Focus Ring

Enhanced focus indicators with a ring effect that appears on focus, providing clear visual feedback for keyboard navigation.

### Validation States

Built-in styling for invalid states via `aria-invalid` attribute. The border and ring change to destructive colors when validation fails.

### Dark Mode

Seamlessly adapts to dark mode with appropriate background colors and contrast adjustments.

## Accessibility

- **Labels**: Always associate the textarea with a label using `htmlFor` and `id` attributes
- **Placeholder**: Don't rely solely on placeholder text for instructions - use labels and helper text
- **Validation**: Use `aria-invalid` and `aria-describedby` to link error messages
- **Required Fields**: Add `aria-required="true"` for required fields
- **Disabled State**: Properly disables keyboard and mouse interaction
- **Focus Management**: Clear focus indicators meet WCAG contrast requirements

## Usage Guidelines

### When to Use

- Multi-line text input (comments, messages, descriptions)
- Longer form fields (feedback, bio, notes)
- Any input where users need to enter multiple sentences or paragraphs

### Best Practices

1. **Sizing**: Provide adequate initial height based on expected input length
2. **Labels**: Always include a visible label for context
3. **Helper Text**: Add helper text for format requirements or character limits
4. **Validation**: Show clear error messages below the textarea
5. **Character Limits**: Display character count when enforcing maxLength
6. **Resize**: The component auto-resizes, but you can set min/max height with utility classes

### Controlled vs Uncontrolled

**Uncontrolled** (simpler, for most cases):
```tsx
<Textarea defaultValue="Initial value" />
```

**Controlled** (when you need to manage state):
```tsx
const [value, setValue] = useState("");
<Textarea value={value} onChange={(e) => setValue(e.target.value)} />
```
