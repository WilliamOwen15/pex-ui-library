An invisible utility component that creates consistent, responsive spacing between elements. Unlike margin and padding utilities, Spacer provides a declarative way to manage whitespace as a standalone element in your layout.

## Basic Usage

The Spacer component creates vertical space by default. It's invisible but takes up space in the layout.

<ComponentPreview
  name="Basic Spacer"
  code={`<div>
  <div className="p-4 bg-slate-100 rounded-lg">

  </div>

  <div className="p-4 bg-slate-100 rounded-lg">
    <p>Second Section</p>
  </div>
</div>`}
>
  <div>
    <div className="p-4 bg-slate-100 rounded-lg">
      <p>First Section</p>
    </div>
    <Spacer size="md" />
    <div className="p-4 bg-slate-100 rounded-lg">
      <p>Second Section</p>
    </div>
  </div>
</ComponentPreview>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"xs" \| "sm" \| "md" \| "lg" \| "xl" \| "2xl" \| "3xl"` | `"md"` | The size of the spacing |
| `axis` | `"horizontal" \| "vertical" \| "both"` | `"both"` | Direction of spacing - vertical creates height, horizontal creates width, both creates a square |
| `responsive` | `{ sm?, md?, lg?, xl? }` | - | Responsive size overrides at different breakpoints |
| `className` | `string` | - | Additional CSS classes to apply |

## Size Scale

| Size | Pixels | Use Case |
|------|--------|----------|
| `xs` | 8px | Minimal spacing between tightly related elements |
| `sm` | 16px | Compact spacing, dense layouts |
| `md` | 32px | Default spacing for most use cases |
| `lg` | 48px | Comfortable spacing between sections |
| `xl` | 64px | Large spacing for major sections |
| `2xl` | 96px | Extra large spacing, hero sections |
| `3xl` | 128px | Maximum spacing for dramatic separation |

## Responsive Breakpoints

The `responsive` prop accepts size values for these breakpoints:
- `sm`: 640px and up
- `md`: 768px and up
- `lg`: 1024px and up
- `xl`: 1280px and up

Example:
```tsx
<Spacer
  size="sm"           // 16px on mobile
  responsive={{
    sm: "md",         // 32px on small screens
    md: "lg",         // 48px on medium screens
    lg: "xl",         // 64px on large screens
    xl: "2xl"         // 96px on extra large screens
  }}
/>
```

## Usage Guidelines

### When to Use Spacer

**Use Spacer when:**
- Creating consistent vertical spacing between unrelated sections
- Separating form groups or major UI elements
- Building layouts where spacing is a first-class design element
- You want responsive spacing without custom CSS
- Building component layouts where margin/padding utilities feel awkward

**Don't use Spacer when:**
- Simple margin/padding utilities would be clearer (e.g., `mt-4`)
- Working with flex/grid gaps (use `gap-*` utilities instead)
- Spacing is purely decorative and inconsistent
- You need pixel-perfect custom spacing (use custom classes instead)

### Best Practices

1. **Consistency**: Stick to the predefined size scale for consistent spacing throughout your application
2. **Semantic Meaning**: Larger spacers should separate less related content
3. **Responsive Design**: Use the `responsive` prop to adjust spacing for different screen sizes
4. **Accessibility**: The component includes `aria-hidden="true"` since it's purely decorative
5. **Axis Clarity**: Be explicit with the `axis` prop when using flex layouts to avoid confusion

### Spacer vs. Margin/Padding

**Choose Spacer when:**
- You want declarative, visible spacing in your JSX
- Spacing needs to be responsive across breakpoints
- Building reusable components where spacing is configurable
- Working with dynamic layouts where spacing might change

**Choose margin/padding when:**
- Simple, one-off spacing needs
- Spacing is tightly coupled to a specific element
- Working with existing utility-first patterns
- Need more control over specific edges (top, bottom, left, right independently)

### Accessibility

- Automatically includes `aria-hidden="true"` as it's purely for layout
- Does not interfere with screen readers or keyboard navigation
- Creates semantic spacing without affecting the accessibility tree

## Performance Notes

The Spacer component is lightweight and uses only CSS for rendering. Responsive classes are generated dynamically based on the `responsive` prop, but this has negligible performance impact. The component does not trigger re-renders unnecessarily.
